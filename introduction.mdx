## 🔐 How It Works

VTULab uses a **single endpoint system**.

You don’t need separate sandbox URLs. Instead:

- `pk_test_...` → Test mode
- `pk_live_...` → Live mode

The API automatically detects your mode from the key.

Example request:

```bash
curl https://api.vtulab.com/v1/identity/nin/verify \
  -H "Authorization: Bearer pk_test_xxxxxxxxx" \
  -H "Content-Type: application/json" \
  -d '{"nin": "12345678901"}'
```
