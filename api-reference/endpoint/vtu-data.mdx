---
title: "POST /v1/vtu/data"
description: "Purchase a data plan."
---

## Authentication

Required: `api.user` + `throttle:api-key`

## Headers

- `Authorization: Bearer <API_KEY>` **or** `X-API-KEY: <API_KEY>`
- `Content-Type: application/json`
- `Accept: application/json`

## Request Schema

| Field | Type | Required | Rules |
| --- | --- | --- | --- |
| `plan_id` | integer | Yes | `required`, `integer`, `exists:data_plans,id` |
| `phone_number` | string | Yes | `required`, `string`, `regex:^\\d{11}$` |

## Sample Request

```json
{
  "plan_id": 12,
  "phone_number": "08012345678"
}
```

## cURL

```bash
curl -X POST "https://api.example.com/v1/vtu/data" \
  -H "Authorization: Bearer <API_KEY>" \
  -H "Content-Type: application/json" \
  -d '{
    "plan_id": 12,
    "phone_number": "08012345678"
  }'
```

## Sample Success Response (200)

```json
{
  "status": true,
  "message": "Data purchase successful",
  "data": {
    "type": "service-dependent object"
  }
}
```

## Sample Error Response (422)

```json
{
  "status": false,
  "message": "Validation failed.",
  "errors": {
    "phone_number": [
      "The phone number format is invalid."
    ]
  }
}
```
