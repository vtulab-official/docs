---
title: "POST /v1/identity/nin/verify"
description: "Verify a NIN record."
---

## Authentication

Required (`Authorization: Bearer <API_KEY>` or `X-API-KEY: <API_KEY>`)

## Request Body

- `nin` (string, required): exactly 11 digits
- `is_consent` (boolean-like, required): must be accepted (`true`, `1`, `on`, or `yes`)

```json
{
  "nin": "12345678901",
  "is_consent": true
}
```

## cURL

```bash
curl -X POST "https://api.example.com/v1/identity/nin/verify" \
  -H "Authorization: Bearer lv_vtulab_sk_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx" \
  -H "Content-Type: application/json" \
  -d '{
    "nin": "12345678901",
    "is_consent": true
  }'
```

## Success Response (200)

```json
{
  "status": true,
  "message": "NIN Verified Successfully",
  "data": {
    "note": "service-dependent object"
  }
}
```

## Error Response (400)

```json
{
  "status": false,
  "message": "Validation failed.",
  "errors": {
    "nin": [
      "The nin field must be 11 digits."
    ]
  }
}
```
