---
title: "POST /v1/identity/nin/verify"
description: "Verify NIN."
---

## Authentication

Required: `api.user` + `throttle:api-key`

## Headers

- `Authorization: Bearer <API_KEY>` **or** `X-API-KEY: <API_KEY>`
- `Content-Type: application/json`
- `Accept: application/json`

## Request Schema

| Field | Type | Required | Rules |
| --- | --- | --- | --- |
| `nin` | string | Yes | `required`, `digits:11` |
| `is_consent` | boolean/string | Yes | `required`, `accepted` |

## Sample Request

```json
{
  "nin": "12345678901",
  "is_consent": true
}
```

## cURL

```bash
curl -X POST "https://api.example.com/v1/identity/nin/verify" \
  -H "Authorization: Bearer <API_KEY>" \
  -H "Content-Type: application/json" \
  -d '{
    "nin": "12345678901",
    "is_consent": true
  }'
```

## Sample Success Response (200)

```json
{
  "status": true,
  "message": "NIN verification successful",
  "data": {
    "type": "service-dependent object"
  }
}
```

## Sample Error Response (422)

```json
{
  "status": false,
  "message": "Validation failed.",
  "errors": {
    "nin": [
      "The nin field must be 11 digits."
    ]
  }
}
```
