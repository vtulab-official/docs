## Base URLs

API route mounting differs by environment:

- **Production:** API is served on the `api.{app.domain}` subdomain.
  - Example: `https://api.example.com`
- **Non-production (local/staging):** API routes are prefixed with `/api`.
  - Example: `https://example.com/api`

Use the same endpoint paths in this documentation, and prepend the correct base URL for your environment.

## Versioning

Current version: `v1`

- VTU endpoints: `/v1/vtu/*`
- Identity endpoints: `/v1/identity/*`

## Public Endpoint

### GET `/`

Returns basic API health/info.

**Authentication:** Not required.

**Headers:**

- `Accept: application/json` (recommended)

**cURL**

```bash
curl -X GET "https://api.example.com/" \
  -H "Accept: application/json"
```

**Success response (200)**

```json
{
  "status": true,
  "message": "API is healthy",
  "data": {
    "service": "api",
    "version": "v1"
  }
}
```

> `data` fields may vary by deployment.

**Error response example (500)**

```json
{
  "status": false,
  "message": "Server error occurred."
}
```
