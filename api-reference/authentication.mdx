---
title: "Authentication"
description: "API key formats, middleware behavior, and authorization failures."
---

Protected endpoints use the `api.user` middleware.

## API Key Headers

Send your API key using **either** header:

- `Authorization: Bearer <API_KEY>`
- `X-API-KEY: <API_KEY>`

## Authorization Outcomes

- Missing key: `401 Unauthorized`
  - `"API Key is missing."`
- Invalid or inactive key: `401 Unauthorized`
  - `"Invalid or inactive API Key."`
- IP not allowed (when key has `ip_whitelist` configured): `403 Forbidden`
  - `"IP address not authorized."`

## Key Mode Detection

The key type sets request mode:

- Live key → `api_mode=live`
- Test key → `api_mode=test`

## JSON Response Enforcement

`ForceJsonResponse` middleware enforces JSON responses for API requests.

## Example (protected endpoint)

```bash
curl -X POST "https://api.example.com/v1/vtu/airtime" \
  -H "Authorization: Bearer <API_KEY>" \
  -H "Content-Type: application/json" \
  -d '{
    "network": "mtn",
    "phone_number": "08012345678",
    "amount": 100
  }'
```
