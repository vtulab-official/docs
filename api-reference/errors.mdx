---
title: "Error Handling"
description: "Standard API response envelope and common failure payloads."
---

All API responses are JSON.

## Success Wrapper

```json
{
  "status": true,
  "message": "string",
  "data": {}
}
```

`data` can be any valid JSON value.

## Error Wrapper

```json
{
  "status": false,
  "message": "string",
  "errors": {}
}
```

`errors` is optional and is usually present for validation failures.

## Validation Error Shape

```json
{
  "status": false,
  "message": "Validation failed.",
  "errors": {
    "field": [
      "The field is required."
    ]
  }
}
```

## Not Found Shape

```json
{
  "status": false,
  "message": "Resource not found."
}
```

## Generic Server Error

Production-safe message:

```json
{
  "status": false,
  "message": "Server error occurred."
}
```

## Common Status Codes

- `400` Bad request
- `401` Unauthorized
- `403` Forbidden
- `404` Not Found
- `429` Too Many Requests
- `500` Internal Server Error
